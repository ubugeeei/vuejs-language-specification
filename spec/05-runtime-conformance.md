# Runtime Conformance

## 1. Scope

The runtime suite is the only JavaScript-specific suite in this repository. It exists to standardize DOM-observable and state-observable behavior that cannot be expressed in a language-neutral way.

The reference runner is executed by Vitest. Local executable runtime test suites in the current repository run in Browser Mode from source-first SFC inputs.
Those suites live under [`testsuites/runtime/`](../testsuites/runtime/) and are organized under numbered domain subdirectories that mirror the second segment of the canonical runtime id while preserving insertion space: `10-dom`, `20-forms`, `30-components`, `40-lifecycle`, and `50-reactivity`.

## 1.1. Runtime Transition Model

Runtime suites in this repository are source-first:

```text
compile_runtime : RuntimeSourceInput → RootComponent
RuntimeSourceInput ::= ⟨kind = "sfc", filename, source⟩
```

The local reference harness compiles the source input with the official Vue SFC compiler before mounting.

Formally:

```text
mount_input(i) ::= mount(compile_runtime(i))
```

Only behavior observable from the compiled-and-mounted source input is normative.

## 1.2. Runtime Transition Model

Runtime test suites are modeled as labeled transitions:

```text
step : RuntimeState × Action → RuntimeState
observe : RuntimeState → Observation
```

For a runtime test suite `t = ⟨input, Σ0, α1..αn, expected⟩`, conformance is:

```text
RuntimePass(I, t) ⇔
  let Σ0 = mount_input(input)
  let Σn = step(...step(step(Σ0, α1), α2)..., αn)
  in ObsEq(observe(Σn), expected)
```

Only the test-suite-defined observation is normative.

## 1.3. Environment Model

The runtime harness distinguishes:

```text
RuntimeEnv ::= BrowserEnv | NodeEnv
BrowserEnv ::= ⟨dom, eventLoop, mountPoint⟩
NodeEnv    ::= ⟨scheduler, reactiveGraph⟩
```

A local runtime test suite declares exactly one `RuntimeEnv` class. Cross-environment equivalence is not implied unless a future test suite explicitly states it.

## 1.4. Observation Model

Runtime observations are drawn from:

```text
DomObs   ::= ⟨text?, html?, props*, attrs*, selectedState*⟩
StateObs ::= ⟨refValues*, emittedEvents*, lifecycleMarks*⟩
Observation ::= DomObs | StateObs | ⟨DomObs, StateObs⟩
```

Only the selected observation fields asserted by the local runtime test suite are normative.

## 2. Observable Surface

A runtime test suite MAY assert:

- text content
- DOM tree shape
- DOM property state
- event-driven state transitions
- provide/inject visibility
- lifecycle side effects that become observable through DOM or state
- selected reactivity invariants

It MUST NOT require matching private scheduler internals unless those internals become externally observable in the test suite.

## 4. Harness Rule

The runtime suite uses a deliberately small harness:

- DOM test suites compile local SFC inputs with the official Vue SFC compiler and mount the resulting component into a fresh container
- the test suite itself performs interaction and observation
- cleanup runs after every test suite

This keeps the runtime suite production-oriented and minimizes harness-specific semantics.

## 5. Coverage Surface

Runtime conformance in this repository is represented by the conjunction of:

- executable local runtime test suites under [`testsuites/runtime/`](../testsuites/runtime/)
- Browser Mode execution under [`verification/runtime-conformance.browser.spec.ts`](../verification/runtime-conformance.browser.spec.ts) for DOM-observable test suites
- vendored `vuejs/core` runtime and reactivity source files under [`provenance/vendor/vuejs-core/packages/runtime-core/`](../provenance/vendor/vuejs-core/packages/runtime-core/), [`provenance/vendor/vuejs-core/packages/runtime-dom/`](../provenance/vendor/vuejs-core/packages/runtime-dom/), and [`provenance/vendor/vuejs-core/packages/reactivity/`](../provenance/vendor/vuejs-core/packages/reactivity/)
- repository-level traceability under [`provenance/traceability/vuejs-core.traceability.pkl`](../provenance/traceability/vuejs-core.traceability.pkl) and [`provenance/traceability/ubugeeei-vize.traceability.pkl`](../provenance/traceability/ubugeeei-vize.traceability.pkl)

An implementation claim for the runtime target MUST execute the local runtime suite. Vendored runtime corpus entries with status `planned` remain within scope for future executable promotion.
